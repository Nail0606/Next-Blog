### 공통 부분

1. Home 로고
    1. 해당 버튼을 누르면 메인 페이지로 와야 함.
2. 네비게이션 바
    1. home, about, posts, contact 가 존재함
    2. home 을 누르면 메인 페이지로 와야 함.
    3. about 을 누르면 About 페이지로 와야 함.
    4. posts 을 누르면 메인 페이지로 와야 함.
    5. contact 를 누르면 Contact 페이지로 이동해야 함.

### 메인 페이지

1. 화면 상단 중앙에 자기 소개가 들어가야 함
    1. 자기 소개는 맨 위 사진, 그 밑 이름, 그 밑 자기소개, 그 밑 Contact Me 버튼으로 이루어짐
    2. contact Me 버튼을 누르면 Contact 페이지로 이동해야 함.
2. 자기소개 밑 부분에 Featured Posts 가 존재해야 함.
    1. 해당 부분에는 본인이 선택한 보여주고 싶은 포스트만 보여주어야 함.
    2. 해당 포스트에는 이미지, 제목, 부제, 카테고리, 작성 날짜가 존재해야 함.
    3. 각각 포스트를 클릭하면 해당 포스트 상세 페이지로 이동함
3. Featured Posts 밑에는 You may like 항목이 존재해야 함.
    1. Featured Posts 를 제외한 모든 Post 가 이곳에 존재해야 함.
    2. 가로로 1줄로 보여주어야 하며, 양 옆의 화살표 버튼으로 슬라이드 되고, 자동으로 슬라이드 되어야 함.
    3. 모든 포스트를 무한적으로 볼 수 있음 ⇒ 재사용

### About 페이지

1. 메인 페이지의 자기 소개가 최상단에 위치해야 함.
2. 그 밑에는 상세한 자기소개가 존재해야 함.

### Contact 페이지

1. 나에게 연락할 수 있는 깃허브, 유튜브 등의 SNS 로고가 존재해야 함
    1. 해당 로고를 클릭하면, 각 링크로 연결되어야 함.
2. 그 밑에는 나에게 연락이 가능한 Form 이 존재해야 함.
    1. 해당 Form 에는 Your Email, Subject, Message 입력 칸이 존재해야 함.
    2. 그 밑의 Submit 버튼이 존재해야 하며, 해당 버튼을 누르면 실제 내 이메일로 전송되어야 함.
    3. 전송 성공 여부에 따라, 성공적으로 전송 / 실패되었다는 알림을 몇초간 보여줘야 함.
    4. 전송이 성공했다면, Form 안의 내용은 삭제되어야 함.

### Posts 페이지

1. 화면 좌측에는 포스트들이 3열로 나열되어 있어야 한다.
2. 화면 우측에는 각 카테고리 항목과 All Posts 카테고리가 존재한다.
3. 기본적으로는 All Posts 로 적용되어야 한다.
4. 적용되는 카테고리의 Post 들을 보여준다.
5. 각 포스트를 클릭하면, 각 Post 상세 페이지로 이동한다.

### Post 상세 페이지

1. URL 링크는 각 포스트의 Id어야 한다.
2. 상세 페이지에는 이미지, 제목, 작성 날짜, description, 본문 내용이 존재해야 한다.
3. 본문 내용은 마크다운 문법을 사용할 수 있어야 한다.(서버에 .md 형식으로 저장)
4. 본문 밑에는 이전, 다음 포스트로 가는 링크가 있어야 함.
5. 맨 밑에는 Footer 가 있어야 한다.
